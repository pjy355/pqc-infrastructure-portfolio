# ROS2 딥러닝 위치추정 QA 분석

다민 로봇팀과 협업하여 ROS2 기반 딥러닝 위치추정(localization) 시스템의 센서 데이터 품질을 분석한 내용이다.

---

## 배경

다민 로봇팀이 개발 중인 딥러닝 보조 위치추정 시스템은 IMU 센서 데이터를 입력으로 받아 로봇의 위치를 추정한다. 이 과정에서 모델 자체의 성능보다 먼저, 입력으로 들어가는 센서 데이터의 품질이 결과에 큰 영향을 준다는 점이 문제로 제기되었고, 이를 QA 관점에서 들여다보는 역할로 협업에 참여했다.

---

## 분석 내용

**IMU 타임스탬프 동기화 문제**
IMU 센서에서 들어오는 데이터의 타임스탬프가 항상 일정한 간격으로 찍히지 않는 경우가 있었다. 50Hz로 동작해야 하는 센서인데, 실제로 타임스탬프 간격을 측정해보니 안정성이 떨어지는 구간이 존재했다. 타임스탬프가 흔들리면 딥러닝 모델이 학습하거나 추론할 때 시퀀스 정보가 왜곡되기 때문에, 이 부분을 먼저 확인하는 것이 우선순위였다.

**enable_web 파라미터의 영향**
ROS2 노드 설정 중 enable_web이라는 파라미터가 센서 데이터의 품질, 특히 타임스탬프 안정성에 영향을 주는지 평가했다. 이 옵션이 켜져 있을 때와 꺼져 있을 때를 비교하면서, 웹 인터페이스 관련 처리가 센서 파이프라인의 실시간성에 부하를 주는지를 확인하는 작업이었다.

**오프라인 평가 시 데이터셋 경로 검증**
모델을 오프라인으로 평가할 때 실제로 올바른 데이터셋을 참조하고 있는지 확인하는 절차도 포함됐다. 경로가 잘못 지정되어 있거나, 런타임에 캡처된 데이터와 평가에 쓰이는 데이터셋이 다른 경우 평가 결과 자체가 의미가 없어지기 때문에, 이 부분을 QA 체크리스트에 넣어 검증했다.

---

## 발견한 점

ROS2 시스템에서는 센서 데이터의 타임스탬프 일관성이 딥러닝 모델의 성능에 직접적인 영향을 미친다. 모델 구조나 하이퍼파라미터를 튜닝하기 전에, 먼저 입력 데이터의 타임스탬프가 안정적으로 들어오고 있는지를 확인하는 것이 선행되어야 한다는 점을 확인했다. 같은 맥락에서 enable_web 같은 부가 기능이 센서 파이프라인에 미치는 영향도 무시할 수 없는 변수였다.

---

## 협업 방식

다민 로봇팀과는 직접 코드를 작성하기보다, 센서 데이터와 타임스탬프를 측정하고 그 결과를 분석해서 팀에 전달하는 QA 역할로 협업했다. 로봇 시스템 자체에 대한 배경지식이 다른 팀과 기술적인 내용을 공유하고, 문제를 비전문가도 이해할 수 있는 형태로 설명하는 경험을 쌓을 수 있었다.
